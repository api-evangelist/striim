# Striim

> Unified data integration and streaming platform offering change data capture (CDC), real-time streaming analytics, and data validation. Striim exposes a token-authenticated REST API (WActionStore queries, system health, Application Management) consumed against your own Striim server/cluster (default port 9080) or the Striim Cloud service.

This llms.txt was generated by API Evangelist from the apis.io catalog entry for Striim; Striim does not publish its own /llms.txt (docs.striim.com is a webhelp system that serves an HTML redirect for every path).

## APIs

- [Striim Application Management REST API](https://striim.stoplight.io/docs/striim-application-management): Create and manage Striim applications (deploy, start, stop, undeploy, drop), execute TQL commands, retrieve monitoring and file-lineage data, run WActionStore queries (GET /wactions/def, GET /wactions/search), and check system health (GET /health). Token-authenticated; runs against your own Striim server/cluster or Striim Cloud.

## Agents

- [Striim MCP AgentLink](https://www.striim.com/press/striim-introduces-mcp-agentlink-for-agentic-applications/): Streams governed, PII-masked read replicas from 100+ sources into MCP-ready zones exposed to AI agents over the Model Context Protocol; provisioned per deployment.
