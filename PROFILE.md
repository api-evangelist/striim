# Striim

Unified data integration and streaming platform offering change data capture (CDC), real-time streaming analytics, and data validation. Exposes a token-authenticated REST API (WActionStore queries, system health, Application Management) consumed against your own Striim instance or Striim Cloud service.

## APIs

- **Striim Application Management REST API** — REST API to create and manage (deploy, start, stop, undeploy, drop) Striim applications, execute TQL commands, retrieve monitoring and file lineage data, plus WActionStore queries (GET /wactions/def, GET /wactions/search) and system health (GET /health). Token-authenticated; endpoints run against your own Striim server/cluster (default port 9080) or Striim Cloud service. (`https://striim.stoplight.io`)

> Striim has a genuine, publicly documented REST API (WActionStore query, system health, Application Management), token-authenticated and exercised against your own Striim instance/cloud rather than a single global public base URL. An OpenAPI-style interactive contract is hosted on Stoplight (v3.10.3), but no standalone downloadable OpenAPI/JSON spec URL was confirmed. Sample code is at github.com/striim/rest-api-samples. Agent-native: Striim's 'MCP AgentLink' is a product/architecture for exposing governed data replicas over MCP to customer-hosted MCP servers — there is NO single Striim-hosted public MCP endpoint, so agent_native is false. No llms.txt and no published agent skills found. The Java event publishing API requires an SDK from Striim support and is not an open REST interface.
